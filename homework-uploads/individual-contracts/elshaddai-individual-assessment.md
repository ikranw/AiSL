# Part A – Individual Contribution 
My primary individual contribution to our Senior Design project, AiSL, was leading much of the backend architecture and systems integration work. Specifically, I focused on building the LLM and retrieval-based text-to-gloss pipeline, which converts user-entered English text into ASL grammatical structure that can later be mapped to avatar animations. This involved designing the backend workflow for text processing, retrieval logic, prompt structuring, and the overall data flow between components. I worked on defining how the system would move from English input → ASL gloss output → frontend display → Unity avatar animation pipeline. In addition to the backend, I also contributed to the technical integration efforts for deployment, supporting the communication pipeline from the backend, to the frontend application, to the Unity avatar system. I also helped translate our Unity WebGL build into a VR-compatible environment. 

This project allowed me to directly build upon the technical skills I identified in my initial assessment from last fall, particularly in Python development, AI/LLM systems, software integration, and user-centered design. I strengthened my competencies in backend architecture, prompt engineering, and cross-platform software integration. One of my biggest successes was helping establish a functioning end-to-end pipeline that supports our live demo: LLM backend to frontend interface to Unity WebGL avatar playback. At the same time, one of the biggest obstacles was working with technologies that required rapid learning and experimentation, especially Unity WebGL integration, VR compatibility, and ensuring reliable communication between software layers. This required significant debugging, iterative testing, and collaboration across technical domains. Through this process, I developed stronger problem-solving skills, learned how to work through integration, and improved my ability to translate high-level system goals into buildable technical workflows.

# Part B – Team Contribution / Group Assessment (Two Paragraphs)
As a group, our team successfully built a working proof-of-concept pipeline and live demo for AiSL, which was our major project milestone. We accomplished the integration of multiple complex systems: a backend LLM-driven text-to-gloss translation structure, a frontend interface for user interaction, and a Unity WebGL avatar capable of signing a limited vocabulary of ASL words and sentence sequences. Reaching a functioning demo that shows the pipeline from user text input all the way to avatar signing output was a major accomplishment and reflects the strength of our collaboration. While the current version is still limited to a constrained sign vocabulary and semi-structured sentence outputs, we have established the technical foundation needed for future work, particularly around automating sign generation and expanding the avatar’s signing capabilities beyond a few predefined words and phrases.   

This project taught me a great deal about group work in highly technical environments. One of the most successful aspects of our teamwork was how we were able to divide responsibilities according to strengths while still supporting one another across workstreams. Each team member contributed significantly in areas that required both prior knowledge and rapid learning of new technologies. I especially want to recognize the work that went into learning and implementing unfamiliar software tools throughout the project lifecycle. Fareena deserves special recognition for her work on frontend development and web deployment, which was critical in making the application accessible and demo-ready. Ikran deserves special recognition for the Unity and avatar implementation, especially the work involved in getting the avatar to physically sign and integrating animation playback. Overall, while my efforts centered more heavily on backend systems and integration, our success came from the fact that each team member took ownership of major technical components and remained collaborative through obstacles, debugging challenges, and evolving project requirements.
